---

### Мета роботи
Спроєктувати архітектуру системи, обрати технологічний стек, описати схему взаємодії компонентів та розробити модель даних (ER-діаграму).

---

### 1. Обраний технологічний стек
* **Frontend:** React.js (Vite). Компонентний підхід дозволяє швидко створювати динамічні таблиці гравців та картки статистики.
* **BaaS (Backend as a Service):** Supabase (PostgreSQL). Надає готову автентифікацію, базу даних та автоматичне REST API «з коробки».
* **Database:** PostgreSQL. Надійна реляційна база для зберігання списків гравців, їхніх номерів та статусів.
* **Security:** Supabase Auth + RLS (Row Level Security). Дозволяє розмежовувати доступ до даних безпосередньо в базі.

---

### 2. Архітектурна модель (C4 Simplified)

#### 2.1 System Context (Контекст)
Користувач (Тренер) взаємодіє з веб-додатком для перегляду статистики, додавання гравців та зміни їхнього статусу здоров'я.

#### 2.2 Container Diagram (Контейнери)
* **Браузер (React SPA):** Відображає інтерфейс, обробляє логіку фільтрації та анімації.
* **Supabase Client (SDK):** Здійснює прямі запити до бази даних через захищений протокол.
* **Supabase Cloud:** Хмарне сховище, що відповідає за авторизацію та збереження таблиць.

---

### 3. Проєктування БД (ER-Model)
Для поточної реалізації обрано спрощену та ефективну модель даних:

**Таблиця profiles (Auth):**
| Атрибут | Тип | Опис |
|:---|:---|:---|
| id | UUID (PK) | Унікальний ідентифікатор користувача |
| email | String | Пошта тренера |

**Таблиця players (Основні дані):**
| Атрибут | Тип | Опис |
|:---|:---|:---|
| id | Int (PK) | Унікальний ID гравця |
| number | Int | Номер на футболці |
| position | String | Амплуа (Воротар, Захисник і т.д.) |
| health_status | String | Стан ("В строю", "Травмований") |
| user_id | UUID (FK) | Посилання на тренера (для RLS захисту) |

---

### 4. Нормалізація
* **1НФ:** Кожен атрибут (номер, позиція) є атомарним.
* **2НФ:** Усі дані гравця залежать від його унікального первинного ключа.
* **3НФ:** Відсутні транзитивні залежності. Наприклад, статус здоров'я залежить безпосередньо від гравця, а не від його позиції.
