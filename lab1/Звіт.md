# Лабораторна робота №1

**Тема:** Веб-додаток для управління футбольним клубом
**Студент:** Кириченко Сергій Васильович
**Група:** 403-ТН

---

### Мета роботи
Сформувати та зафіксувати функціональні та нефункціональні вимоги до вебдодатка для управління складом футбольної команди, визначити ключових користувачів та технологічний стек проєкту.

---

### 1. Зацікавлені сторони (Stakeholders)
* **Тренер (Coach):** Головний користувач. Йому важливо бачити актуальний список гравців, їхні номери, позиції та стан здоров'я для формування складу на гру.
* **Адміністратор:** Керує доступом до системи, забезпечує актуальність бази даних гравців.
* **Розробник:** Використовує React та Supabase для створення швидкого та надійного SPA-додатка (Single Page Application).

---

### 2. Функціональні вимоги (FR)

| ID | User Story (Користувацька історія) | Критерії прийнятності |
|:---|:---|:---|
| **FR-01** | Як тренер, я хочу реєструватися та входити в систему. | Використання Supabase Auth (Email/Password). |
| **FR-02** | Як тренер, я хочу додавати нового гравця до списку. | Поля: номер, позиція (вибір зі списку). |
| **FR-03** | Як тренер, я хочу бачити загальну статистику команди. | Автоматичний підрахунок кількості здорових та травмованих. |
| **FR-04** | Як тренер, я хочу змінювати статус гравця ( 🚑 / ✅ ). | Миттєве оновлення статусу "В строю" або "Травмований". |
| **FR-05** | Як тренер, я хочу фільтрувати гравців за позиціями. | Кнопки для відбору: Воротарі, Захисники, Півзахисники, Нападники. |
| **FR-06** | Як тренер, я хочу редагувати дані гравця. | Можливість змінити номер або амплуа існуючого запису. |
| **FR-07** | Як тренер, я хочу видаляти гравців, що покинули клуб. | Кнопка швидкого видалення ( 🗑️ ) без зайвих підтвержень. |

---

### 3. Нефункціональні вимоги (NFR)
* **NFR-01 (Продуктивність):** Час відображення списку гравців після входу не повинен перевищувати 500 мс.
* **NFR-02 (Безпека):** Використання політик RLS (Row Level Security) у базі даних для захисту даних кожного тренера.
* **NFR-03 (Зручність):** Адаптивний інтерфейс для роботи зі смартфонів безпосередньо на футбольному полі.
* **NFR-04 (Візуалізація):** Використання анімацій для підтвердження успішності дій (плавна поява рядків).
* **NFR-05 (Надійність):** Зберігання даних у хмарній базі PostgreSQL (Supabase).

---

### 4. Обмеження і припущення
* Додаток реалізується як Client-side застосунок (без окремого Node.js сервера).
* Весь обмін даними йде напряму між React та Supabase SDK.
* Для роботи потрібне стабільне інтернет-з'єднання.

---

### Висновок
У ході ЛР1 розроблено концепцію веб-додатка для управління футбольним клубом. Визначено, що основним пріоритетом є зручність тренера в управлінні складом. Обраний стек (React + Supabase) дозволяє реалізувати всі вимоги без зайвих витрат на серверне адміністрування.
