### Мета роботи
[cite_start]Сформувати та зафіксувати функціональні та нефункціональні вимоги до вебдодатка для управління складом футбольної команди, визначити ключових користувачів та технологічний стек проєкту. [cite: 6]

---

### 1. Зацікавлені сторони (Stakeholders)
* **Тренер (Coach):** Головний користувач. [cite_start]Йому важливо бачити актуальний список гравців, їхні номери, позиції та стан здоров'я для формування складу на гру. [cite: 8]
* [cite_start]**Адміністратор:** Керує доступом до системи, забезпечує актуальність бази даних гравців. [cite: 9]
* [cite_start]**Розробник:** Використовує `React` та `Supabase` для створення швидкого та надійного SPA-додатка (Single Page Application). [cite: 10]

---

### 2. Функціональні вимоги (FR)

| ID | User Story (Користувацька історія) | Критерії прийнятності |
|:---|:---|:---|
| **FR-01** | Як тренер, я хочу реєструватися та входити в систему. | [cite_start]Використання `Supabase Auth` (Email/Password). [cite: 12] |
| **FR-02** | Як тренер, я хочу додавати нового гравця до списку. | [cite_start]Поля: номер, позиція (вибір зі списку). [cite: 12] |
| **FR-03** | Як тренер, я хочу бачити загальну статистику команди. | [cite_start]Автоматичний підрахунок кількості здорових та травмованих. [cite: 12] |
| **FR-04** | Як тренер, я хочу змінювати статус гравця ( 🚑 / ✅ ). | [cite_start]Миттєве оновлення статусу "В строю" або "Травмований". [cite: 12] |
| **FR-05** | Як тренер, я хочу фільтрувати гравців за позиціями. | [cite_start]Кнопки для відбору: Воротарі, Захисники, Півзахисники, Нападники. [cite: 12] |

| **FR-07** | Як тренер, я хочу видаляти гравців, що покинули клуб. | [cite_start]Кнопка швидкого видалення ( 🗑️ ) без зайвих підтверджень. [cite: 12] |

### 3. Нефункціональні вимоги (NFR)
* [cite_start]**NFR-01 (Продуктивність):** Час відображення списку гравців після входу не повинен перевищувати `500 мс`. [cite: 14]
* [cite_start]**NFR-02 (Безпека):** Використання політик `RLS` (Row Level Security) у базі даних для захисту даних кожного тренера. [cite: 15]
* [cite_start]**NFR-03 (Зручність):** Адаптивний інтерфейс для роботи зі смартфонів безпосередньо на футбольному полі. [cite: 16]
* [cite_start]**NFR-04 (Візуалізація):** Використання анімацій для підтвердження успішності дій (плавна поява рядків). [cite: 17]
* [cite_start]**NFR-05 (Надійність):** Зберігання даних у хмарній базі `PostgreSQL` (Supabase). [cite: 18]

---

### 4. Обмеження і припущення
* [cite_start]Додаток реалізується як Client-side застосунок (без окремого `Node.js` сервера). [cite: 20]
* [cite_start]Весь обмін даними йде напряму між `React` та `Supabase SDK`. [cite: 21]
* [cite_start]Для роботи потрібне стабільне інтернет-з'єднання. [cite: 22]
