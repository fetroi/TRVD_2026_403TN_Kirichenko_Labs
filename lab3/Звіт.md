### Мета роботи

### 1. Реалізація Data Access Layer (DAL)
[cite_start]Взаємодія з базою даних `PostgreSQL` реалізована через офіційний `Supabase SDK`[cite: 74]. [cite_start]Це дозволило відмовитися від проміжного Node.js сервера на користь прямого та безпечного доступу до даних[cite: 75].

[cite_start]**Основна сутність — Таблиця `players`:** [cite: 76, 77]
| Поле | Опис |
|:---|:---|
| `id` | Унікальний ідентифікатор |
| `number` | Номер на футболці |
| `position` | Амплуа (позиція) |
| `health_status` | Статус готовності |
| `user_id` | Зв'язок з тренером (власником запису) |

[cite_start]**Методи доступу (реалізовані в `App.jsx`):** [cite: 79]
* [cite_start]`getPlayers()`: Отримання списку гравців з сортуванням за номером[cite: 80].
* [cite_start]`handleAddPlayer()`: Додавання нового запису або оновлення існуючого[cite: 81].
* [cite_start]`deletePlayer()`: Видалення запису за ідентифікатором[cite: 82].
* [cite_start]`toggleStatus()`: Миттєва зміна статусу здоров'я[cite: 83].

### 2. Логіка обробки даних (Logic Layer)
[cite_start]Вся бізнес-логіка зосереджена в обробниках подій React-компонента[cite: 85]:
* [cite_start]**Валідація:** Перевірка наявності номера гравця перед відправкою (`if (!newNumber) return`)[cite: 86].
* [cite_start]**Адаптація статусів:** Автоматична зміна стану між "В строю" та "Травмований"[cite: 87].
* [cite_start]**Синхронізація:** Виклик оновлення стану після кожної CRUD-операції для актуальності даних[cite: 88].

---

### 3. Взаємодія з API (Supabase REST)
[cite_start]Додаток використовує автоматично згенероване REST API[cite: 90]. Статус-коди відповідей:

| Код | Значення | Опис |
|:---|:---|:---|
| `200` / `201` | **OK / Created** | [cite_start]Успішне зчитування або додавання даних [cite: 91] |
| `400` | **Bad Request** | [cite_start]Помилка у форматі даних (наприклад, текст замість номера) [cite: 92] |
| `401` | **Unauthorized** | [cite_start]Доступ без активної сесії (JWT) [cite: 93] |
| `403` | **Forbidden** | [cite_start]Спроба доступу до чужих даних (RLS) [cite: 94] |

### 4. Обробка помилок та безпека
* [cite_start]**Error Handling:** Використано блоки `try...catch...finally` для обробки винятків та керування станом завантаження (`loading`)[cite: 96].
* [cite_start]**JWT Auth:** Кожен запит автоматично підписується токеном користувача з `localStorage` через SDK[cite: 97].
