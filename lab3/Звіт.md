**Група:** 403-ТН

### Мета роботи
Реалізувати логіку взаємодії з даними (Data Access) та інтегрувати клієнтську частину з хмарною базою даних Supabase. Налаштувати структуру таблиць та забезпечити валідацію даних при виконанні CRUD-операцій.

---

### 1. Реалізація Data Access Layer (DAL)
Взаємодія з базою даних PostgreSQL реалізована через офіційний Supabase SDK. Це дозволило відмовитися від проміжного Node.js сервера на користь прямого та безпечного доступу до даних.

**Основна сутність:**
* **Таблиця players:** Зберігає анкети гравців.
* **Поля:** id, number (номер на футболці), position (амплуа), health_status (статус готовності), user_id (зв'язок з тренером).

**Методи доступу (реалізовані в App.jsx):**
* **getPlayers():** Отримання списку гравців з сортуванням за номером.
* **handleAddPlayer():** Додавання нового запису або оновлення існуючого.
* **deletePlayer():** Видалення запису за ідентифікатором.
* **toggleStatus():** Миттєва зміна статусу здоров'я.

---

### 2. Логіка обробки даних (Logic Layer)
Вся бізнес-логіка зосереджена в обробниках подій React-компонента, що забезпечує високу швидкість відгуку інтерфейсу:
* **Валідація:** Перед відправкою даних у базу перевіряється наявність номера гравця (if (!newNumber) return).
* **Адаптація статусів:** Логіка toggleStatus автоматично змінює стан гравця між "В строю" та "Травмований" без участі зовнішніх сервісів.
* **Синхронізація:** Після кожної зміни (додавання, видалення, редагування) викликається функція оновлення стану, що гарантує актуальність даних на екрані.

---

### 3. Взаємодія з API (Supabase REST)
Додаток використовує автоматично згенероване REST API від Supabase. Кожен запит супроводжується статус-кодами:
* **200 OK / 201 Created:** Успішне зчитування або додавання гравця.
* **400 Bad Request:** Помилка у форматі даних (наприклад, нечислове значення номера).
* **401 Unauthorized:** Спроба доступу до даних без активної JWT-сесії.
* **403 Forbidden:** Спроба редагування чужих даних (контролюється політиками RLS).

---

### 4. Обробка помилок та безпека
* **Error Handling:** Використано конструкції try...catch та блоки finally для коректного відображення станів завантаження (loading) та виведення повідомлень про помилки.
* **JWT Auth:** Кожен запит до бази даних автоматично підписується токеном користувача, який Supabase SDK витягує з localStorage.
